# 이슈 트래커: 로컬 마크다운

이 프로젝트의 이슈와 스펙(PRD라고도 함)은 `.scratch/` 디렉토리의 마크다운 파일로 관리됩니다.

## 규칙

- 기능별 디렉토리: `.scratch/<기능-슬러그>/`
- 스펙 파일: `.scratch/<기능-슬러그>/spec.md`
- 구현 이슈 파일: `.scratch/<기능-슬러그>/issues/<번호>-<슬러그>.md` (01부터 시작, 단일 파일 금지)
- 트라이아지 상태: 각 이슈 파일 상단에 `Status:` 라인으로 기록 (라벨 문자열은 `triage-labels.md` 참조)
- 댓글과 대화 이력은 파일 하단에 `## Comments` 섹션에 추가

## 스킬이 "이슈 트래커에 게시하라"고 할 때

`.scratch/<기능-슬러그>/` 아래에 새 파일을 생성 (필요시 디렉토리도 생성).

## 스킬이 "관련 티켓을 가져오라"고 할 때

지정된 경로의 파일을 읽습니다. 사용자가 일반적으로 경로나 이슈 번호를 직접 전달합니다.

## 탐색 작업

`/wayfinder`에서 사용됩니다. **지도(map)** 는 티켓당 하나의 **자식(child)** 파일을 가진 파일입니다.

- **지도**: `.scratch/<노력>/map.md` — 노트 / 결정사항 / 미해결 사항 본문
- **자식 티켓**: `.scratch/<노력>/issues/번호-<슬러그>.md` (01부터 시작), 본문에 질문 포함. `Type:` 라인에 티켓 유형 기록 (`research`/`prototype`/`grilling`/`task`); `Status:` 라인에 `claimed`/`resolved` 기록
- **차단**: 상단에 `Blocked by: 번호, 번호` 라인. 나열된 모든 파일이 `resolved`일 때만 차단 해제
- **탐색 대상**: `.scratch/<노력>/issues/`에서 열려 있고, 차단되지 않았으며, 선언되지 않은 파일 스캔; 번호 순서대로 우선 처리
- **선언**: 작업 전에 `Status: claimed` 설정 후 저장
- **해결**: `## Answer` 섹션에 답변 추가, `Status: resolved` 설정, 지도의 결정사항에 컨텍스트 포인터(요약 + 링크) 추가
